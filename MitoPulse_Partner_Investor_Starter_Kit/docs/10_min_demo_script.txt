
MITOPULSE — 10 MINUTE DEMO SCRIPT

Minute 0–1 — The Problem
Explain that the largest fraud today is social engineering.
Examples:
- Family scams
- Marketplace scams
- Phishing
- Deepfake voice attacks

Key message:
Current systems verify identity.
They do NOT verify if an interaction should happen.

Minute 1–3 — The Idea
Introduce the core concept:

MitoPulse does not ask:
“Who are you?”

MitoPulse asks:
“Should this interaction happen?”

Core components:
- Relational graph
- Behavior memory
- Guardian nodes
- Risk propagation

Minute 3–5 — Pilot Network
Show the pilot network with 4 nodes:

nd_manu_01
nd_ana_01
nd_bruno_01
nd_carla_01

Explain:
The system learns patterns and detects anomalies.

Minute 5–7 — Guardian Layer
Explain institutional guardians:

guardian_bank
guardian_telco
guardian_marketplace

Institutions help supervise trust decisions.

Minute 7–9 — Fraud Propagation
If a node is compromised:
risk propagates through the graph.

This reveals coordinated fraud clusters.

Minute 9–10 — Vision
MitoPulse can become the trust layer of the internet.

Cloudflare protects web traffic.
Stripe protects payments.

MitoPulse protects human interactions.
