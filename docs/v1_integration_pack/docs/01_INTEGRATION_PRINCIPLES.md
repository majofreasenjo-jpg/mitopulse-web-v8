# Principios de Integración

## 1. Overlay, no reemplazo
MitoPulse se conecta a lo que ya existe:
- core banking
- antifraude
- ERP
- SCADA / historian
- TMS / WMS
- CRM / ticketing
- SIEM / logs
- data lake / warehouse

## 2. Canonical model
Todo se traduce a:
- nodes
- edges
- events
- states
- time
- labels
- external_fields

## 3. Industry adapters
Cada industria mantiene sus protocolos, pero MitoPulse los consume vía adaptadores.

## 4. RFDC-first
El modelo canónico alimenta RFDC. La IA ayuda a normalizar, clasificar y explicar.

## 5. Outputs in client language
La salida siempre se adapta al lenguaje operativo del cliente.
