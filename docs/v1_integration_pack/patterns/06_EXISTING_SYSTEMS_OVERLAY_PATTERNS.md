# Existing Systems Overlay Patterns

## Pattern A — Banking Overlay
Existing stack:
- core banking
- fraud / AML
- device intelligence
- BI / data lake

MitoPulse overlay:
- consume transfers, accounts, devices, beneficiaries
- construir red dinámica
- calcular GSI + índices bancarios
- entregar recomendaciones al fraud ops / AML

---

## Pattern B — ERP / SAP-like Overlay
Existing stack:
- finance
- inventory
- procurement
- costs
- planning
- maintenance
- sales

MitoPulse overlay:
- consume materials, PO, suppliers, assets, cost signals
- conectar supply + finance + operations
- detectar propagación de presión
- generar indicadores de continuidad / supply stress

---

## Pattern C — Energy Overlay
Existing stack:
- SCADA
- historian
- maintenance
- trading
- dispatch
- weather / market feeds

MitoPulse overlay:
- mapear unidades, tanques, rutas, contratos
- detectar cascadas y criticalidad
- activar continuity recommendations

---

## Pattern D — Logistics Overlay
Existing stack:
- TMS
- WMS
- telematics
- route planning
- ERP
- port/vessel data

MitoPulse overlay:
- mapear envíos, rutas, nodos de congestión
- detectar delay propagation
- optimizar bottlenecks y resiliencia de red
