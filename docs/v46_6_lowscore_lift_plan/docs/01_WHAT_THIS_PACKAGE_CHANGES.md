# Qué cambia realmente

El paquete anterior definía el plan.
Este paquete da un paso más: cambia la clase matemática de varios mecanismos.

## Antes
- varios módulos usaban score composition simple
- forecasting corto con escalamiento lineal
- calibración y benchmarking como roadmap
- aprendizaje real como esquema conceptual
- integración real como overlay conceptual

## Ahora
- HRM no lineal y por dominio
- hazard y time-to-criticality explícitos
- wave propagation con damping y amplification terms
- calibration dataset schema formal
- benchmark protocol reproducible
- learning loop con feedback schema
- integration QA schema y score de completitud
