# 05. Notas para robustecer la patente

La invención NO debe presentarse como:
- usar grafos para fraude
- usar ML para fraude
- detectar clusters sospechosos

La invención SÍ debe presentarse como:
- infraestructura de confianza relacional
- evaluación pre-contacto
- quorum relacional
- trazas estigmérgicas
- señales de daño digital
- reserva alostática de confianza
- morfogénesis de fraude

Esto aumenta la distancia conceptual respecto a graph analytics bancario tradicional.
